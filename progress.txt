### Key Codebase Patterns

**TDD Structure**:
- Features: BDD format (Gherkin) in features/ (unit/, integration/, e2e/)
- Step definitions: tests/steps/ and tests/e2e/steps/
- Production code: src/
- Run tests: npm run test:unit, npm run test:e2e, npm run test

**System Architecture**:
- Systems: Encapsulated logic classes (ActionWheel, BeadBag, CombatResolver, etc.)
- Entities: Domain objects (Token, CharacterToken, MonsterToken, Entity)
- Services: Stateful managers (CharacterStorageService, DataLoader)
- UI Components: Phaser graphics classes (HeroSelectionBar, SelectedHeroPanel, BattleUI)
- Scenes: Game scenes (MenuScene, BattleScene, CharacterCreationScene)

**Data Patterns**:
- YAML config in public/data/
- TypeScript interfaces for all data (CharacterData, WeaponDefinition, etc.)
- localStorage for character persistence
- Immutable updates where possible

**Testing Patterns**:
- Each feature file is focused (one system/concern)
- Given/When/Then step structure
- Mock/stub external dependencies
- E2E tests verify integration scenarios
